0.1
骨架 + 单 Agent 对话
Agent · LLM · Tool · Trace · Chat · Server · CLI
当前阶段 基建
目标：跑通最基础的 chat，验证内核选型。

做什么

  • src/ 项目骨架
  • LangGraph 内核（Master Agent 单节点 graph）
  • LLMPort + 1 个 adapter（OpenAI 或 DeepSeek）
  • ToolPort + 内置 tool（read_file / write_file / shell）
  • TracerPort + JSONLTracer
  • CheckpointerPort + LangGraph SqliteSaver
  • Server（HTTP/SSE）最简 chat API
  • CLI：sigma chat

不做

  • Task 系统
  • Skill 系统
  • Sub-agent
  • RAG / Memory
  • 多 provider 路由

退出标准

sigma chat 能跟 LLM 多轮对话
工具调用流跑通（agent 调 read_file 读文件回答）
Trace JSONL 文件能看到每次 LLM 调用 / tool call
中断恢复：kill 进程后 sigma chat --resume <session> 能续上
0.2
Task 引擎
Task · Chat↔Task
基建
目标：Chat / Task 双视图跑通，Task 基建完整可用。

做什么

  • Task Engine：状态机（queued → running → completed / paused / failed / cancelled）
  • Task queue + SQLite 持久化
  • Pause/resume 机制（LangGraph interrupt + Command(resume=...)）
  • Chat ↔ Task 打通：升级建议 + 完成回流
  • CLI：sigma task new / list / view / pause / resume / cancel

不做

  • 周期性 task（cron）
  • Multi-agent
  • RAG / Memory
  • Coding agent

退出标准

sigma task new "帮我总结这篇文章" 能后台跑 task
Task 卡住能进 paused，sigma task resume 能续
Chat 中建议升级 task，task 完成后回流通知
Trace 里能完整看到 task 执行（含 pause / resume）
0.3
Memory + Context + Self-Improvement MVP
Memory · Context · Improvement
基建
目标：让 Sigma 有记忆、会记偏好——"越用越懂你"的起点。

做什么

  • Memory 三层存储：global / session / task
  • Memory CLI：sigma memory list / edit / delete
  • Context Engine：多源拼装 + token budget 管理
  • Self-improvement MVP：显式偏好提取 → memory 写入 → 下次生效
  • 用户可查看 / 编辑 / 删除 memory 条目

不做

  • RAG
  • 隐式信号提取（V2+）
  • Audio metadata

退出标准

用户说"以后回答用中文"，下次新 session 自动用中文
sigma memory list 能看到提取的偏好，可编辑
隔天开新 session 还记得偏好
长对话（50+ 轮）context 压缩正常，不崩不截断
0.4
RAG + 讨论书场景
RAG · Context 增强
场景 3 ✓
目标：Sigma 能基于外部文档回答问题，场景 3（讨论书）端到端跑通。

做什么

  • RAG 多 index 管理（每个 index 独立）
  • Indexing pipeline：parse → chunk → embed → store
  • 至少 1 个 vector store（Qdrant / Chroma / LanceDB）
  • 检索带引用（章节 / 页码）
  • Context Engine 增强：RAG 结果参与 token budget
  • CLI：sigma rag create / index / search / list / delete
  • 场景 3 端到端跑通

不做

  • Multi-agent（单 agent + RAG tool 即可）
  • 代码库索引优化

退出标准

上传一本书 → 基于内容多轮问答，回答带引用
讨论 50 轮后总结笔记，质量可用
多个 index 共存互不干扰
RAG 检索结果正确参与 token budget 分配

路线图原则

1
跟着 4 个核心场景走——任何 milestone 如果不能让某个核心场景前进，就不做
2
学习价值优先——选实现路径时优先选能学到东西的，不优先选最简的
3
不预先做 Port——只在已有 ≥ 2 个真实需求时才做抽象
4
小步走——每个 milestone 可独立 demo / 独立写博客复盘
5
先跑通再打磨——每个场景先端到端跑通，再优化细节
